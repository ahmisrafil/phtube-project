Q1. Discuss the scope of var, let, and const.

Answer: In JavaScript, var, let, and const are used for variable declaration and have different scopes and behaviors. Understanding their scope is crucial for writing clean and predictable code.


Var:

  • Var has function-level scope, meaning it is only accessible within the function where it's declared or globally if declared outside of any function.
  • Variables declared with var are hoisted to the top of their containing function or global scope, which means they are initialized with undefined before the code execution.
  • It's generally considered less safe and less predictable compared to let and const because of its hoisting and lack of block-level scoping.

Let :

  • Let has block-level scope, meaning it's only accessible within the block (e.g., inside loops, conditionals, and functions).
  • Variables declared with let are also hoisted but are not initialized, so they remain in a "temporal dead zone" until they are declared.
  • Unlike var, redeclaring a variable with let in the same scope is not allowed and results in a syntax error.

Const :

  • Const also has block-level scope, just like let.
  • Variables declared with const cannot be reassigned after initialization. However, the value they hold can still be mutated if it's an object or an array.
  • Like let, redeclaring a variable with const in the same scope is not allowed and results in a syntax error.

Q2. Tell us the use cases of null and undefined.

Answer: In JavaScript, null and undefined are two distinct values that are often used for different purposes. Understanding their use cases is important for writing clean and error-free code.


Undefined :

'undefined' represents a variable that has been declared but has not been assigned a value yet. It's also the default value of function parameters that have not been provided with an argument.


Use Case :

  • When you declare a variable without assigning it a value, it's automatically initialized to undefined.
  • When you access an object property that does not exist, JavaScript returns undefined.
  • Function parameters that are not provided with arguments are also undefined.


Null :

'null' is a deliberate value that represents the absence of any object value or no value at all. It's often used when you want to explicitly indicate that a variable should have no value.


Use Case :

  • Clearing a variable or object property.
  • Checking if an object property has been intentionally set to a non-value state.
  • As a return value to indicate the absence of a meaningful result or object.

Q3. What do you mean by REST API?.

Answer: REST API, which stands for Representational State Transfer Application Programming Interface, is a set of architectural principles and constraints used for designing and interacting with web services. REST is not a protocol but rather a style of designing networked applications, primarily based on the principles outlined by Roy Fielding in his doctoral dissertation in 2000.

Key characteristics and principles of RESTful APIs include:


Stateless: Each request from a client to a server must contain all the information needed to understand and process the request. The server does not store any client state. This simplifies server design and allows for better scalability.

Client-Server: REST separates the client and server, allowing them to evolve independently. Clients are not concerned with data storage, while servers are not concerned with user interfaces. This separation promotes scalability and flexibility.

Uniform Interface: RESTful APIs have a consistent and uniform interface that is easy to understand and use. This uniformity is achieved through a set of well-defined constraints, including- Resources, HTTP Methods and Representation.

Resource-Based: REST treats resources as the core abstraction, and each resource is identified by a unique URL. Resources can represent real-world entities (e.g., a user, a product) or abstract concepts (e.g., a session, a configuration).
